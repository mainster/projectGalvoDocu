DAC_WP_t DAC_SetDualChanSigned ( int16_t  Data2,
int16_t  Data1 
)

#include <adc_dac_dma.h>

Dual DAC set channel wrapper function.

This function extends the DAC_SetDualChannelData function which is a part of CMSIS driver layer. The extension provides active range checks for FIXME: min max valuesprovided by a serial command to protect connected Galvos against algorithm failures which can raise in overcurrent states.

Attention
DO NOT ACCESS THIS FUNCTION VIA DIRECT FUNCTION CALL !!! WRITE ACCESS TO DAC REGISTERS SHOULD BE HANDLED BY FUNCTION POINTER DAC_SecureSetDualChanSigned.
Parameters
[in]Data2Raw DAC compensator algorithm output value for channel 2.
[in]Data1Raw DAC compensator algorithm output value for channel 1.
Returns
"Fingerprint" to check whether the ASG system has triggered the fuse or not!

DAC_WP_t(* DAC_SecureSetDualChanSigned) (int16_t, int16_t)

#include <adc_dac_dma.h>

Definition of DAC Write Access pointer.

Use this function pointer to provide unprivileged, secure write access to the DAC hardware. When the ASG enters the system into fault state, the pointer reference changes to the ASG-tripped function" DAC_SetDualChanSigned_Tripped from which all actuator control signals are forced to output a predefined, safe value.

Parameters
[in]RawDAC compensator algorithm output value for channel 2.
[in]RawDAC compensator algorithm output value for channel 1.
